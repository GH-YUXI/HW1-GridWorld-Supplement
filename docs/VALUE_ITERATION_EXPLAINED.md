# Value Iteration 與隨機策略說明

## 核心概念

Value Iteration 是一種用來求解 Markov Decision Process（MDP）最佳策略的方法。
它會反覆更新每個狀態的價值函數 `V(s)`，直到收斂，再依據價值函數挑出最佳動作。

## Value Iteration 更新公式

```math
V_{k+1}(s)=\max_a \left[r(s,a,s') + \gamma V_k(s')\right]
```

其中：

- `s`：目前狀態
- `a`：可採取動作
- `s'`：下一狀態
- `r`：即時回饋
- `γ`：折扣因子

## 隨機策略 Policy Evaluation

隨機策略不是取最大值，而是對所有動作的價值取期望值：

```math
V_{k+1}^{\pi}(s)=\sum_a \pi(a|s)\left[r(s,a,s') + \gamma V_k^{\pi}(s')\right]
```

本專案的隨機策略為：

```text
上、下、左、右各 25%
```

「執行隨機策略」會直接顯示一次抽樣軌跡；「逐步播放隨機策略」會把同樣概念改成動畫播放，讓使用者看到每一步隨機選方向、移動或留在原地的過程。

## 本專案中的環境設定

- 動作集合：`上、下、左、右`
- 撞牆或撞障礙物：停在原地
- 每執行一次動作都得到 `stepReward`，包含進入終點的那一步
- 終點本身價值固定：`V(goal)=0`
- 折扣因子限制：`0 ≤ γ < 1`
- 若起點無法到達終點，系統會先提示，不執行策略計算

## 同分時隨機選擇最佳動作

在某些狀態中，可能有多個動作都得到相同的最佳值，例如：

```text
Up    -> -3.2
Right -> -3.2
Down  -> -4.1
Left  -> -5.0
```

此時最佳動作集合為：

```text
[Up, Right]
```

本專案會從這個集合中隨機選一個顯示在畫面上，因此每次執行時，箭頭可能不同，但都仍屬於最佳策略。
